What happens at 5:00 PM Monday

⚙️ End-to-end flow
  1. Cron fires on the Hermes side → ~/.hermes/scripts/laundry_trycents_weekly.sh runs.
  2. That wrapper delegates to laundry_flow.sh in your Laundry TryCents project directory.
  3. The flow checks whether Chrome on Windows is reachable at localhost:9222 (Chrome DevTools Protocol).
    • If yes → the script uses the existing browser.
    • If no → it launches start_chrome_cdp.bat via cmd.exe, then waits up to 30 seconds for CDP to become available.
  4. The Python CDP driver runs the order flow end-to-end and stops on the Order Summary page (checkout review), so you can verify line items before submitting.
  5. You manually click Submit in your Chrome window when you are satisfied with the order.
Implementation note: laundry_flow.sh probes $CDP_URL/json/version (default http://localhost:9222), starts Windows Chrome if needed, then runs _cdp_driver.py. See the project README.md for file roles and environment variables.

Important caveats

Machine must be awake and unlocked

At 5:00 PM Mondays your Windows machine must be awake and unlocked. If it is asleep or hibernating, the job fails because Chrome cannot render the UI reliably.

Mitigations:

  • Windows Settings → Power → set Sleep: Never while plugged in, or
  • Task Scheduler: a “wake the computer” task at 4:55 PM Mondays (or similar).
WSL must be running

A normal Hermes daemon keeps WSL alive. If WSL is fully shut down, cron does not run. Check with wsl --status from Windows when troubleshooting.

Chrome profile conflicts

Chrome already running with a different --user-data-dir can conflict with the CDP instance. The cleanest approach is to keep the CDP Chrome profile (for example C:\chrome-cdp) in a separate window from your daily browsing profile.

Natural-language control

💬 Via your assistant

Instead of typing Hermes commands, you can say things like:

  • “Run the laundry job now” — triggers a manual run through the same path.
  • “Pause the laundry job” — temporarily disables the schedule.
  • “Cancel it” — removes the cron job (destructive; confirm intent).
